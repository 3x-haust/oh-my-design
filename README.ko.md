# oh-my-design

[English](./README.md) | **한국어**

[![GitHub Stars](https://img.shields.io/github/stars/3x-haust/oh-my-design?style=flat-square)](https://github.com/3x-haust/oh-my-design/stargazers)

**학습 데이터의 평균이 아니라, 사람처럼 디자인한다.**

AI가 만든 인터페이스는 고장 나는 일이 거의 없다. 고장보다 나쁘다 — *정확한데 익명이다*.
인디고에서 바이올렛으로 흐르는 그라디언트. 똑같은 그림자를 얹은 기능 카드 세 장.
전부 가운데 정렬. 로켓 이모지로 시작하는 제목. "Unlock the power of."

버그가 아니다. 관점 없는 모델이 내놓는 필연적인 결과다 — 지금까지 본 모든 것의 평균값.
`omd`는 이 실패를 **측정 가능하게** 만들고, 그 위에서 사람 디자이너가 실제로 도는 루프를
돌린다: 문제를 의심하고, 컨셉을 정하고, 실물 레퍼런스를 연구하고, 하나만 제대로 짓고,
렌더된 걸 눈으로 보고, 본 것이 문제 정의를 다시 쓰게 한다.

---

## 🚀 설치

Claude Code 플러그인으로 (스킬이 `omd:ultradesign`, `omd:scout` 형태로 표시된다):

```
/plugin marketplace add 3x-haust/oh-my-design
/plugin install omd@oh-my-design
```

요구사항: Node ≥ 22.18. 첫 `omd render` 실행 때 Playwright가 헤드리스 Chromium을
자동으로 설치한다.

| 스킬 | 트리거 예시 | 하는 일 |
|:--|:--|:--|
| `omd:ultradesign` | "디자인해줘", "이거 리디자인", "landing page 만들어줘" | 루프 전체를 끝까지. 승인 요청 없음 — 블로그를 시키면 블로그가 나오고, 모든 결정과 이유가 기록으로 남는다. |
| `omd:scout` | "레퍼런스 수집해줘", "비슷한 사이트 연구해줘" | 측정된 레퍼런스 보드만 만든다 — 전체 페이지, 컴포넌트, 타이포그래피 스터디, 모션 스터디, 커뮤니티 스레드. 디자인은 하지 않는다. |

| `omd:humanize` | "AI티 빼줘", "번역투 없애줘", "사람 말처럼 고쳐줘" | 아무 글이나 AI 문체 틱을 뺀다. 한국어·영어 번역투, 기계적 나열, 상투어, 균일한 리듬, 헤징 더미를 제거하되 사실은 한 글자도 바꾸지 않는다. |
| `omd:coach` | "뭘 반복해서 틀리지?", "검사 이력 보여줘" | 검사 이력을 읽는다. 뭘 반복해서 틀리는지, 뭐가 나아지는지, 다음에 뭘 볼지. 데이터가 얇으면 추세를 지어내길 거부한다. |

## 🤔 이게 뭔가

[DesignPref](https://arxiv.org/abs/2511.20513)에서 전문 디자이너 스무 명이 UI 쌍
12,000개를 비교 평가했다. 서로 얼마나 일치했는가 — **Krippendorff's α = 0.248.**
비교의 4분의 1 이상에서 의견이 거의 완전히 갈렸다. "좋은 디자인"의 보편적 보상 함수는
존재하지 않는다. 그러니 평균 취향에 최적화한 모델이 위의 익명한 결과물을 내놓는 건
사고가 아니라 *목적함수의 논리적 귀결*이다.

더 좋은 모델을 요구하는 건 이 문제의 해법이 아니다. 익명성은 가중치의 버그가 아니라
평균 취향을 향해 정확히 작동하는 가중치의 결과다. 대신 만들 수 있는 것:

1. **평균을 탐지한다.** 슬롭은 검사 가능하다. 결정론적 규칙 일곱 개 — 관점 있는
   작업에서는 침묵하고 익명한 작업에서만 발화한다.
2. **관점을 강제한다.** 컨셉 은유 하나가 모든 결정을 거른다. 신뢰 가능한 회계사는
   튀지 않는다. 새벽 3시 편의점은 여백을 두지 않는다.
3. **실물에서 배우되, 안전하게.** 스크린샷이 아니라 측정값 — 간격 사다리, 타입 스케일,
   모션 지속시간. 레퍼런스를 눈으로 본 모델은 그걸 그대로 복사한다(Jansson & Smith,
   1991). 측정값을 읽은 모델은 그것으로부터 배운다.
4. **본다.** 헤드리스로 렌더하고 숫자를 계산한다. 대비비나 패딩 값을 눈대중하게 두지 않는다.
5. **색과 타입을 이론으로 근거한다.** 컨셉 단계에서 내장 이론 팩을 읽고 도메인 리서치를
   돌린 뒤 방향을 정한다. 결정에는 이유가 붙는다. 감각이 아니라.

## 🔁 파이프라인

```
                         ┌────────────────────────────────────────────────┐
                         │                                                ▼
  ① FRAME ──▶ ② CONCEPT ──▶ ③ REFERENCE ──▶ ④ COMMIT ──▶ ⑤ BUILD ──▶ ⑥ SEE ──▶ ⑦ REFRAME
   문제를        이론 팩 +      omd-scout이      구조 하나,     omd-hand가    렌더하고     본 것이
   의심한다      도메인 리서치   실물을           비용 명시      짓는다        측정한다     프레임을
                → 방향 정함     측정한다                                               다시 쓴다
```

`omd-framer`가 브리프를 심문한다. `omd-scout`이 레퍼런스 보드를 만든다 — 절대 복사하지 않고,
항상 측정한다. `omd-hand`가 커밋된 구조 하나를 짓는다. `omd-eye`는 새 컨텍스트에서 비평한다:
작업을 만든 이유를 모르기 때문에 그 이유를 방어할 수 없다.

## 🧹 슬롭 린터

`omd check`는 대비·히트 영역·간격·토큰 커버리지, 그리고 **슬롭** — 평균으로 수렴한
작업의 서명 — 을 계산한다. 전부 warn이다. 각 규칙은 의도된 선택에 대해 틀릴 수 있고,
기각하려면 이유를 글로 남겨야 한다.

| 규칙 | 잡는 것 |
|:--|:--|
| `SLOP-GRADIENT` | 인디고→바이올렛 그라디언트 (헥사 목록이 아니라 색상환 대역으로 판정) |
| `SLOP-RADIUS-MONOCULTURE` | 모든 모서리가 같은 반경 — 재질의 위계가 없다 |
| `SLOP-SHADOW-MONOCULTURE` | 같은 그림자의 반복 — 전부 떠 있으면 아무것도 안 떠 있다 |
| `SLOP-EVERYTHING-CENTERED` | 강조가 아니라 기본값이 된 가운데 정렬 |
| `SLOP-EMOJI-HEADING` | 타이포그래피가 못 한 일을 대신하는 이모지 |
| `SLOP-COPY` | "Unlock the power of…", "no fluff here", 한국어 AI 문체 틱 |
| `SLOP-TRIPLE-CARD` | 똑같은 기능 카드 세 장 — 무엇이 중요한지 아무도 정하지 않았다는 고백 |
| `SLOP-LEAKED-RATIONALE` | 설계 근거가 배송된 카피에 그대로 인용됨 |

마지막 규칙이 핑크 엘리펀트 실패다: *"잡다한 거 넣지 마"*라고 하면 모델은 페이지에
*"잡다한 내용은 없습니다"*라고 적는다. `omd check`는 페이지 텍스트와 프로젝트 설계
기록 사이에 연속 5어절이 겹치면 결정론적으로 잡아낸다.

실물로 보정했다: 규칙들은 *정확하고 접근성도 통과하지만 익명인* 픽스처에서 전부
발화하고, 같은 내용에 관점이 있는 픽스처에선 침묵하며, linear.app을 슬롭이라 부르지
않는다.

`omd check`는 발견이 있으면 exit 1 — 그대로 **CI 디자인 린터**로 쓸 수 있다.

## 📚 이론 팩

색은 제품에 대한 주장이고, 이론 팩은 그 주장의 근거가 사는 곳이다. 파일 일곱 개가
플러그인에 `core/theory/`로 내장된다:

| 파일 | 답하는 질문 |
|:--|:--|
| `color.md` | 도메인별 색 관습과 이유(핀테크·헬스케어·교육·푸드·개발자 도구·럭셔리); 조화 스킴과 선택 시점; 60-30-10 배분; 채도의 레지스터 신호; 배경 온도 |
| `typography.md` | 모듈러 스케일 비율이 정보 거리에 대해 말하는 것; 페어링 이론과 대비의 축; 타입 의미론(서체가 글자 읽히기 전에 말하는 것); 최적 행길이; 행간 시스템 |
| `layout.md` | 게슈탈트 원리의 UI 결정 번역; 시각 위계 도구 우선순위; F·Z 스캔 패턴; 그리드와 리듬; 폰 레스토프 효과와 one memorable thing |
| `motion.md` | Nielsen 1993 응답시간 연구 기반 지속시간 임계값; 이징 의미론(ease-out과 ease-in-out이 각각 말하는 것); 안무 규칙; 모션 예산 |
| `components.md` | 컴포넌트별 결정 가이드: 버튼 위계 상한, 폼 검증 타이밍, 내비게이션 패턴, 테이블, 모달과 그 대안, 토스트 위계, 검색 |
| `craft.md` | 이론서가 건너뛰는 실무 지식: 여러 겹 그림자, 보더 없는 구분, 투명도 계층 텍스트, 옵티컬 정렬과 밸런스, 복합 hover 상태, 60fps 안전 속성 |
| `expressive.md` | 어워드 사이트 해부학: Awwwards 심사 배점과 거기서도 유저빌리티가 크리에이티비티를 이기는 이유; 타이포그래피 히어로; 내러티브로서의 스크롤; 절제 조항이 붙은 기법 카탈로그 |

각 항목은 조건→선택→이유 구조로 쓰여 있다. 일반론 나열이 아니다. 출처는 실명 — Elliot &
Maier, Bringhurst, Müller-Brockmann, Nielsen, NN/g. 컨셉 단계는 방향을 정하기 전에 이
파일들을 읽고, hand는 보드가 커버하지 않는 결정에서 이걸 읽는다.

## 🏗 아키텍처

```
src/
  agents/                       모든 에이전트의 원본
    framer.agent.yaml
    scout.agent.yaml
    hand.agent.yaml
    eye.agent.yaml
  skills/                       모든 스킬의 원본
    omd-ultradesign/SKILL.md
    omd-scout/SKILL.md
    omd-critique/SKILL.md
    omd-humanize/SKILL.md
    omd-coach/SKILL.md
core/
  theory/                       디자인 이론 팩 (dist/에 복사됨)
    color.md
    typography.md
    layout.md
    motion.md
  ref/                          레퍼런스 측정 및 저장
  render/                       헤드리스 Playwright 렌더러
  rules/                        슬롭 린터 엔진
adapters/
  build.ts                      agents/, skills/, dist/, .mcp.json 생성
dist/
  claude/                       Claude Code 직접 설치 (생성됨)
  codex/                        Codex CLI 설치 (생성됨)
agents/                         생성됨 — 직접 편집 금지
skills/                         생성됨 — 직접 편집 금지
.omd/                           프로젝트별 디자인 기록
  frame.md                      현재 문제가 무엇이라고 믿는지
  decisions.md                  이 제품에 왜 초록색이 없는지
  refs/*.json                   측정된 레퍼런스 + 원리 문장
  history.jsonl                 모든 검사 실행 — omd:coach가 읽는 것
```

`npm run build`가 `src/`에서 `agents/`, `skills/`, `dist/`, 루트 `.mcp.json`을 재생성한다.
생성된 디렉터리는 절대 직접 편집하지 말 것.

## CLI

```
omd check  <page> [--json] [--category slop]   린트: a11y, 일관성, 슬롭. 발견 시 exit 1
omd render <page> -o shot.png                  헤드리스 스크린샷 (찍고 실제로 봐라)
omd ir     <page>                              렌더된 DOM → 측정된 노드 트리
omd ref    add|list|show|principles|distance   레퍼런스 보드
omd frame  set|show|reframe|generator          문제 기록 (아무도 서명하지 않는다. 루프가 다시 쓴다)
omd decision "what" --why "why"                후임자가 고마워할 이유 파일
omd coach                                      반복되는 약점, 정직한 추세
```

## 정직한 한계

- 루프의 규율은 프롬프트가 강제하고, *측정*은 코드가 강제한다. 모델은 조언을 무시할 수
  있다 — 대비비를 조작할 수는 없다.
- 슬롭 규칙은 휴리스틱이다. 브랜드가 진짜 보라색일 수 있다. 그래서 warn이고, 기각에
  서면 이유가 필요하다.
- 레퍼런스 캡처는 공개 페이지를 실제 브라우저로 읽어 **측정값과 추론만 저장한다.
  에셋은 절대 저장하지 않는다.** 연구하는 사이트를 존중하라.
- Codex CLI 지원은 트리에 있지만(bare 설치) Claude Code 경로만큼 실전 검증되지 않았다.
