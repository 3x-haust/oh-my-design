# Scenario 02 — 현장 설비 점검 기록 (public regression fixture)

건물 설비 점검 기사 12명이 쓰는 점검 기록 도구를 만들어줘. 기사는 현장에서 휴대폰으로 쓴다:
오늘 배정된 점검지 목록 확인 → 점검지 선택 → 항목별 점검 결과 입력(정상/이상/보류 + 메모)
→ 이상 항목은 사진 없이도 우선순위와 조치 요청을 남김 → 점검 완료 제출. 제출 전에 빠진
항목이 있으면 어디가 빠졌는지 바로 보여야 해. 지하실에서는 네트워크가 자주 끊겨서, 입력한
내용이 사라지면 안 돼. 사무실에서는 관리자(2명)가 데스크톱으로 완료된 점검 결과를 검토해.

## Core user tasks (the grader walks these, in order)

1. 390px에서 오늘 배정 목록을 확인하고 다음 점검지를 고른다.
2. 점검 항목 10개 중 7개를 입력하고 제출을 시도한다 — 빠진 3개 항목이 지목되는지,
   입력한 7개가 보존되는지 확인한다.
3. 이상 항목 1건에 우선순위와 조치 요청을 남긴다.
4. 오프라인 상태 표시(또는 로컬 보존 안내)를 확인한다.
5. 1280px 관리자 화면에서 완료된 점검을 열어 이상 항목만 걸러 본다.

## Invariants

- Use `omd-ultradesign` (oh-my-design). 생성 후 결과물 코드를 손으로 수정하지 않는다.
- 모바일이 1차 뷰포트다: 데스크톱 레이아웃의 축소가 아니라 모바일 태스크 동선이 기준.
- 부분 입력 상태(10개 중 7개)가 별도 상태로 설계되어야 한다.
