# Scenario 03 — 사진 스튜디오 예약 관리: 온보딩 + 설정/결제 (public regression fixture)

1인 사진 스튜디오 예약 관리 서비스의 신규 가입 온보딩과 설정 화면을 만들어줘. 신규 사용자는
가입 직후: 스튜디오 이름/주소 입력 → 촬영 상품 1개 이상 등록(이름·소요시간·가격) → 예약
가능 시간대 설정 → 첫 예약 링크 발급까지 끝내야 서비스를 쓸 수 있어. 기존 사용자는 설정에서
상품·시간대·결제 수단(카드 등록, 플랜 변경, 영수증)을 관리해. 온보딩을 중간에 끄면 다음
로그인 때 이어서 할 수 있어야 하고, 플랜 해지는 실수로 누르기 어렵게 + 되돌릴 수 있게 해줘.

## Core user tasks (the grader walks these, in order)

1. 온보딩 4단계를 순서대로 완료한다 — 각 단계에서 현재 위치와 남은 단계가 보이는지 확인.
2. 2단계에서 잘못된 가격(음수)을 입력하고 오류 안내와 값 보존을 확인한다.
3. 3단계에서 온보딩을 이탈했다가 돌아와 이어서 진행한다.
4. 설정에서 상품 하나를 수정·저장하고, 저장 여부 피드백을 확인한다.
5. 플랜 해지 흐름에 진입해 확인 단계와 복구(해지 취소) 경로를 확인한다.
6. 390px에서 온보딩 1단계와 설정 진입을 반복한다.

## Invariants

- Use `omd-ultradesign` (oh-my-design). 생성 후 결과물 코드를 손으로 수정하지 않는다.
- 온보딩은 wizard 문법(진행 표시·단계별 탈출구·값 보존), 설정은 settings 문법(의도별
  그룹, 저장 피드백, 파괴적 행동 격리)을 따라야 한다.
- 신규 사용자(온보딩)와 재방문 사용자(설정)의 경험이 달라야 한다.
