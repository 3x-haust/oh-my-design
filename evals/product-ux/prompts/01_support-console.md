# Scenario 01 — 지원팀 티켓 콘솔 (fixed development task)

우리 스타트업 고객지원팀(4명)이 쓸 티켓 콘솔을 만들어줘. 이메일/채팅으로 들어온 문의가
티켓으로 쌓여. 팀원은 하루 종일 이 화면에서 산다: 새 티켓 확인 → 검색/필터(상태·채널·담당자)
→ 티켓 열어 대화 확인 → 답장 작성/저장 → 상태 변경(대기/진행/완료). 잘못 완료 처리한 티켓을
되돌리는 일이 잦아. 모바일에서도 급한 답장을 보낼 수 있어야 해.

## Core user tasks (the grader walks these, in order)

1. 첫 화면에서 3초 안에 "무엇부터 처리해야 하는지"를 찾는다.
2. 상태·담당자 필터와 검색으로 특정 티켓을 찾는다. 결과 0건 상태를 확인한다.
3. 티켓을 열고 답장을 작성해 보낸다. 빈 답장 제출로 오류 상태를 확인하고, 입력값이
   보존되는지 확인한다.
4. 티켓을 완료 처리한 뒤 즉시 되돌린다(실행취소). 완료 목록에서 재오픈한다.
5. 390px 뷰포트에서 1–4를 반복한다.

## Invariants

- Use `omd-ultradesign` (oh-my-design). 생성 후 결과물 코드를 손으로 수정하지 않는다.
- 프레임 surface 분류는 `product`가 되어야 하고, 첫 뷰포트의 지배 앵커는 티켓 큐 자체다.
- 상태 증거: loading / empty(첫 실행) / filtered-to-zero / error / success 각각이
  렌더 또는 프로브로 증명되어야 한다.
